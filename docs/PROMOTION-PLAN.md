# Plainvoice — promotion action plan

Solo, low-cost, slow-growth. **Two primary channels: directory listings + communities.**
Everything else accrues underneath. Failure mode isn't the wrong channel — it's
spreading thin and burning out. Sequence it; don't run all five at once.

**Message discipline (everywhere):** lead with **"actually free · unlimited · no
signup · won't rug-pull you."** That gets the click and the recommendation. The AI
features are revenue, not the hook — they sell themselves *inside* the product once
someone's already invoicing. Never lead promotion with AI, or you blur the one thing
that makes you different from the tools you're replacing.

**Status note:** the free core is live and launch-ready now. Lemon Squeezy store
approval (real AI payments) is still pending — but it does NOT block promotion,
because you don't lead with AI.

## Phase 0 — Foundation (THIS WEEK, before any launch)
Make the link worth sharing. A launch with a blank preview wastes the one shot.
- [ ] **Social share image (og:image)** — the biggest gap. Every listing / PH / Show HN
      / Reddit share renders a preview card; right now it's text-only. Need a 1200×630.
- [ ] **3–4 real screenshots** for listing galleries (editor, a finished invoice/PDF,
      the Pricing Promise section, the photo-import flow).
- [ ] **2–3 short demo GIFs** of the wow moments (photo→draft, chat→invoice, language switch).
- [ ] **Submit sitemap** in Google Search Console (`https://plainvoices.com/sitemap-index.xml`).
- [ ] One-liner + 50-word + 250-word descriptions written once, reused across all listings
      (kept in this repo — see "Canned copy" below once drafted).

## Phase 1 — Directory listings (PRIMARY CHANNEL #1 · highest leverage, lowest effort)
These are where humans *and* AI answer-engines pull "best free invoice app" from.
Being listed = starting to become the answer. Mostly free submissions.
- [ ] **AlternativeTo** — list as an alternative to **FreshBooks, QuickBooks, Invoice2go,
      Wave**. This is the flagship: your whole angle is "alternative to the apps that
      rug-pulled you." (alternativeto.net)
- [ ] **SaaSHub** — add product + list as alternative to the same four.
- [ ] **Capterra / GetApp** (same network) — vendor listing.
- [ ] **G2** — product profile.
- [ ] **Product Hunt** — *one* well-timed launch (see Phase 2; don't burn it early).
- [ ] Smaller: **Slant**, **Toolfinder/Uneed**, indie "free tools" lists as you find them.

## Phase 2 — One launch salvo (after domain + og:image are sorted)
One-shot, high-signal, zero cash. Lead with the anti-rug-pull, indie-built story.
- [ ] **Show HN: Plainvoice — free, local-first invoicing; AI is the only thing you pay for**
      — Hacker News rewards the "built from frustration" framing + technical local-first
      angle. Post Tue–Thu morning US time; reply to every comment for the first hours.
- [ ] **Product Hunt launch** — prep gallery (screenshots + GIFs), tagline, first comment
      telling the origin story. Tue–Thu. Line up a few honest "hunters"/supporters.
- [ ] **Indie Hackers** post — same story, you're already in that community.
- [ ] Do NOT do PH and Show HN the same day — stagger by a few days so you can be present
      for each.

## Phase 3 — Communities (PRIMARY CHANNEL #2 · ongoing, sustainable)
Be the honest builder, not the marketer. People constantly ask
"FreshBooks got too expensive — free alternative?"
- [ ] Watch **r/freelance, r/smallbusiness, r/Entrepreneur** + freelance Facebook groups.
- [ ] Answer *real* threads genuinely → disclose you built it → lead with
      "unlimited free, no signup." Reddit punishes pitching, rewards the underdog story.
- [ ] Cadence: a few genuine replies/week. Never copy-paste; never first-to-pitch.
- [ ] Keep a running note of recurring questions → they become Phase 4 SEO pages.

## Phase 4 — Thin bottom-funnel SEO (long game, 1 page at a time — NOT a content machine)
Target people already leaving a paid tool ("get me out of here" intent). These convert.
Build as blog posts / landing pages, grounded in the real product (real screenshots,
the actual Pricing Promise, the photo-import flow) so they read first-hand, not generic.
- [ ] "Free FreshBooks alternative (no subscription, unlimited invoices)"
- [ ] "QuickBooks invoicing alternative — no monthly fee"
- [ ] "Free invoice generator — no signup"
- [ ] "Plainvoice vs FreshBooks" (honest comparison)
- [ ] Ship one good page at a time; this compounds over months underneath everything else.

## Phase 5 — Short-form demo clips (when you ship something demoable)
Three features demo themselves. 30-sec screen captures, captioned, low production.
- [ ] **Photo import** — "I snapped my old paper invoice and it rebuilt it in 3 seconds."
- [ ] **Chat-to-invoice** — dictate an invoice in one sentence.
- [ ] **Overdue chaser / auto-translate** — the tone-perfect / localized moment.
- [ ] Post to **YouTube Shorts / TikTok / X**. One clip per shipped feature, not a schedule.

## Phase 6 — Light owned-audience loop (don't betray the no-signup ethos)
- [ ] Public **changelog** page (e.g. reuse the blog) + an *optional, clearly voluntary*
      "get product updates" email. Low frequency. Trust > aggressive funnel.

---

## Cadence
- **Week 1:** Phase 0 foundation only.
- **Week 2:** Phase 1 listings + open with ONE launch (Show HN *or* PH).
- **Week 3:** the other launch; settle in.
- **Ongoing:** Phase 3 communities (primary) + one Phase 4 SEO page at a time + a Phase 5
  clip whenever you ship something demoable. Let Phases 4–6 accrue; don't force them.
